Detailed Description

dynamic_bitset declaration and implementation.

Standalone file, does not depend on other implementation files or dependencies other than the standard library, can be taken and used directly.

Can optionally include and use libpopcnt if DYNAMIC_BITSET_NO_LIBPOPCNT is not defined and __has_include(<libpopcnt.h>) is true.

Remarks
Include multiple standard library headers and optionally libpopcnt.h.

The size comparison is necessary for the comparison operators to keep their properties. For example without the size comparison the "<=" operator (defined for "A <= B" by "!(B < A)") would no longer be antisymmetric (if A <= B and B <= A, then A == B) because operator==() compare the dynamic_bitset as a container and not a number. For example with bitsets A(0011) and B(011), without the size comparison B < A would be false, A <= B would be true, B <= A would be true, but A == B would be false, breaking the antisymmetric property of the operator. Thus, to respect the properties of the operators, the size is used as a secondary criteria for the comparison of dynamic_bitset which content represent the same number. Therefore, for the previous example with bitsets A(0011) and B(011), B < A is true, A <= B is false, B <= A is true and A == B is false.

If comparing bitsets A and B with the content of A representing the number a, and the content of B representing the number b, this operator would work as follow:

if(a == b)
{
return A.size() < B.size();
}
else
{
return a < b;
}
Remarks
The empty dynamic_bitset is the "lowest" of all bitset and for 0-only bitsets comparison, the shortest is the lowest.

Extract a dynamic_bitset from a character stream using its string representation.

The string representation expected is the same as if generated with dynamic_bitset::to_string() with default parameter, using '1' for true bits and '0' for false bits. On success the content of bitset is cleared before writing to it. The extraction starts by skipping leading whitespace then take the characters one by one and stop if is.good() return false or the next character is neither _CharT('0') nor _CharT('1').
